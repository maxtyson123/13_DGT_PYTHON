Quiz Game 0.54
A Python programming language based application designed to train players for a quiz. The program uses the OpenTDB API to generate questions for the players to practice their quiz skills. The program uses the Requests module to send HTTP requests to the trivia website and then utilizes a self-built renderer and game module to generate a quiz with a simple user interface. After the game, players can track their progression of skills through stats such as time spent. The program allows for the addition of bots to play alongside the player and can be configured to set the accuracy of their AI. The program is flexible and can expand menu sizes and is written to handle unexpected user input and corrupted JSON files.

◆ wait_for_players()

def game.Game.wait_for_players (   self) 
Creates a new server and sets it to the backend of this class. If there are no players then the user is asked
to set the host player, otherwise whatever player is at index 0 will be set to the host. A menu is then
shown, showing all the players currently in the game, it refreshes itself every 3 seconds via manipulation
of the time_limit in the Menu class. When the host decides to start the game any old unconnected users will
be removed from the game and the game will start. The server will then sync the game data with the clients
and play() will be called.
